# 方案：预测与赔率挂钩（市场隐含概率融合）

> 目标读者：实现者（GLM）。本文件是**设计方案**，不是代码。给出数据源、数学、融合点、代码落点、配额与验收标准，按此实现即可。

## 0. 为什么挂钩赔率

博彩市场赔率聚合了"聪明钱"，是公认**校准最好**的概率估计。更关键的一点：足球赔率是 **1X2 三路盘（主胜/平/客胜）**，**自带平局概率 P(draw)** —— 这正是我们模型缺的（我们靠泊松众数推平局，市场化做不出真实 25-30% 的平局率）。所以挂钩赔率能**同时**改善胜负校准 **和** 修复平局真实度。

与现有"可信度建立信任"的产品理念一致：赔率是强可信度信号，理由页展示"市场隐含概率"很有说服力。

## 1. 数据源

**首选 [The Odds API](https://the-odds-api.com/)**：免费档 500 请求/月，覆盖足球（含世界杯），返回多家博彩公司的**十进制赔率 1X2**，JSON，需免费注册拿 key。

- key 走环境变量 `ODDS_API_KEY`（**不要硬编码**；`.gitignore` 已忽略 `.env`）。无 key → 赔率功能关闭，回退纯模型。
- 端点形如：`GET /v4/sports/soccer_fifa_world_cup/odds/?regions=eu&markets=h2h&oddsFormat=decimal&apiKey=...`
  - 一次请求**返回所有已开盘的 WC 比赛**（含各家 bookmaker 的 h2h 三路赔率）。一次拿全，配合长缓存控配额（见 §6）。
- 备选：Pinnacle（更"sharp"但需账号）、其它聚合源。**实现成可插拔适配器**（一个 `fetch_odds()` 接口），方便换源。
- 覆盖局限（必须诚实处理）：只有**已开盘**的近期比赛才有盘口；早期轮次、淘汰赛未定队没有 → **逐场判断，有则用、无则回退模型**。

沿用现有模块风格（`live_data.py`/`squad_data.py`/`form_data.py`）：抓取 + 内存缓存 + 失败返回 `{}` 优雅降级。

## 2. 去水（移除博彩抽水 vig / overround）

十进制赔率 `(oH, oD, oA)` → 原始隐含概率 `1/o`。三者之和 > 1（超出部分=抽水）。归一即去水：

```
rawH, rawD, rawA = 1/oH, 1/oD, 1/oA
over = rawH + rawD + rawA          # overround，通常 1.05~1.12
pH, pD, pA = rawH/over, rawD/over, rawA/over   # 归一后 sum=1，即市场隐含真概率
```

- 多家 bookmaker：对每家先去水得到 (pH,pD,pA)，再**跨家取均值**（或取最低 overround 那家=最 sharp）。记录 `books_n`。
- （可选进阶：Shin/power 去水法更准，先用比例法即可。）

输出结构建议：`{ pairing_key: {"pH":..,"pD":..,"pA":..,"books_n":n,"updated":iso} }`，`pairing_key = frozenset((home_en, away_en))`（队名用 `wc_data` 内部英文键，需做 The Odds API 队名 → 内部键的映射表，类似 `DE_TO_EN`）。

## 3. 融合方案（核心）

**在概率层融合（推荐）**，因为要用到 P(draw)。分三步：

### 3.1 取"模型 1X2"
复用现有泊松联合分布（`predictor._likely_scoreline` 已经在算双方独立泊松的联合网格）。把网格按 i>j / i==j / i<j 求和，得到**模型自己的** `(mH, mD, mA)`：
```
新增 predictor.model_1x2(home, away, played_per_team) -> (mH, mD, mA)   # sum=1
（lam_h/lam_a 用现有 win_prob→期望进球那套；网格上限 max_goals=8 复用）
```

### 3.2 与赔率融合
```
若该对阵有赔率(且 books_n 足够)：
    λ = ODDS_W            # 对赔率的信任权重，默认 0.7（"赔率不骗人"→可偏高）
    pH = (1-λ)*mH + λ*oddsH
    pD = (1-λ)*mD + λ*oddsD
    pA = (1-λ)*mA + λ*oddsA
    (已各自 sum=1，线性组合后仍 sum=1)
否则：
    (pH,pD,pA) = (mH,mD,mA)   # 回退纯模型
```
- `λ` 做成模块常量，可按"距开赛时间/盘口数量"动态调（越临近、盘口越多→越信赔率）。先用固定 0.7。

### 3.3 用融合后的 1X2 驱动展示
- **胜率条**：主胜率 = `pH`，客胜率 = `pA`（或按需归一展示）。
- **预测比分**：先按 `argmax(pH,pD,pA)` 定结果方向；
  - 若为"平" → 取最可能平局比分（联合网格里 i==j 的众数，通常 1-1）；
  - 若为"主胜/客胜" → 取该方向下的众数比分（网格里对应区域 argmax）。
  - 这样**平局率自动贴近市场**（odds 的 pD 对均势局约 0.28），顺带修复了平局真实度。
- **小组赛积分榜 / 淘汰赛签表**：继续走"单一确定性场景"，比分由上面同一套 1X2 决定 → **首页/小组赛/签表仍同场一致**（保持现有不变量）。
- **蒙特卡洛概率**：随机模拟时，对有赔率的场，用融合后的 (pH,pD,pA) 做三项抽样决定胜平负，再在该结果内按泊松采样具体比分；无赔率场维持现状。

> 关键不变量（实现时务必回归）：①首页/小组赛同场比分一致 ②有赔率场的预测方向应与市场一致 ③平局率回到 ~20-28% ④无赔率/无 key 时与当前行为完全一致。

## 5. 诚实 / 准确率复盘的处理（重要）

- **复盘页（`_build_accuracy`）保持纯模型口径**：继续用 `predicted_scoreline(t1,t2)`（不含赔率），因为：①免费档拿不到历史/封盘赔率，②避免"用赛前市场结果反推已踢比赛"的口径混乱。即：**实时预测用赔率，复盘回测不用赔率**，并在页面注明这一区别。
- 若想做"赔率 vs 真实"的对照（很有看点）：需在比赛开赛前**快照当时赔率并落盘存储**（赛后比对）。这是二期，需要持久化（文件/SQLite），本期不做。

---

## 6. 配额与缓存（避免超免费额度）

- 免费 500 请求/月 ≈ **每天 ~16 次**。**一次请求返回全部已开盘比赛** → 只需一个长缓存：
  - `ODDS_CACHE_TTL = 7200`（2 小时）→ 每天 12 次、每月 ~360 次 < 500，安全。
  - 赔率在临近开赛才明显波动；2 小时缓存对预测足够。
- 与现有 `_VIEW_CACHE`(5min) 解耦：视图重建时若赔率缓存未过期直接复用，不发请求。
- 无 key / 超额 / 失败 → 返回 `{}`，全程回退纯模型（与现状一致）。

## 7. 测试（离线、不触网）

新增 `test_odds.py`：
- 去水：给定 `(oH,oD,oA)` 验证归一后 sum≈1、比例正确（含 overround 计算）。
- 跨家均值：多家合成赔率 → 均值正确。
- 队名映射：The Odds API 名 → 内部键。
- `blend_1x2`：合成 odds + 合成 model_1x2，验证 λ 线性组合、sum=1；无赔率回退纯模型。
- 平局修复：构造均势对阵 + 市场高 pD → 预测结果为平局。
- 降级：`odds={}` 时 `blend_1x2 == model_1x2`、`win_prob` 与现状一致。

## 8. 分期落地

1. **一期（本方案）**：`odds_data` + `model_1x2` + `blend_1x2` 概率层融合 + 理由展示 + 测试 + 2h 缓存降级。实时预测挂钩赔率、平局回归真实。
2. **二期（可选）**：开赛前赔率快照落盘 → 复盘页加"模型 vs 市场 vs 真实"三方对照；λ 按临近度动态化；Shin 去水。

## 附：关键公式速查
```
去水:      p_i = (1/o_i) / Σ(1/o_j)
模型1X2:   对泊松联合网格 P(i,j)=Pois(i;λh)·Pois(j;λa) 按 i>/=/<j 求和
融合:      p = (1-λ)·model + λ·odds   (λ=ODDS_W=0.7, 各自sum=1→结果sum=1)
比分:      方向=argmax(pH,pD,pA)；比分=该方向网格区域的众数(平局多为1-1)
胜率条:    home=pH, away=pA
```
